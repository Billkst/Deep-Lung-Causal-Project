# 硬件适配与性能优化指南

## 概述

本文档提供对比算法实验平台在单卡 RTX 3090 (24GB) 环境下的硬件适配和性能优化指南。

## 硬件要求

### 最低配置
- **GPU**: NVIDIA RTX 3090 (24GB VRAM) 或同等性能显卡
- **CPU**: 8 核心或更多
- **内存**: 32 GB RAM
- **存储**: 50 GB 可用空间

### 推荐配置
- **GPU**: NVIDIA RTX 4090 (24GB VRAM) 或更高
- **CPU**: 16 核心或更多
- **内存**: 64 GB RAM
- **存储**: 100 GB SSD

## 性能优化策略

### 1. GPU 显存监控

使用 `GPUMemoryMonitor` 实时监控显存使用情况：

```python
from src.baselines.utils import GPUMemoryMonitor

# 创建显存监控器
monitor = GPUMemoryMonitor(warning_threshold_gb=20.0)

# 在训练循环中检查显存
for epoch in range(epochs):
    # 训练代码...
    monitor.check_memory(context=f"Epoch {epoch+1}")

# 获取统计信息
stats = monitor.get_memory_stats()
print(f"峰值显存使用: {stats['peak_memory_gb']:.2f} GB")
```

**上下文管理器用法**：

```python
with GPUMemoryMonitor(warning_threshold_gb=20.0) as monitor:
    model.fit(X_train, y_train)
    monitor.check_memory(context="Training Complete")
```

### 2. 梯度累积

当显存不足时，使用梯度累积技术模拟大批量训练：

```python
from src.baselines.utils import GradientAccumulationTrainer

# 创建梯度累积训练器
trainer = GradientAccumulationTrainer(
    model=model,
    optimizer=optimizer,
    accumulation_steps=4  # 有效批量大小 = batch_size × 4
)

# 训练循环
for epoch in range(epochs):
    for batch_idx, (X_batch, y_batch) in enumerate(dataloader):
        # 前向传播
        output = model(X_batch)
        loss = criterion(output, y_batch)
        
        # 反向传播（带梯度累积）
        trainer.backward_step(loss, batch_idx)
    
    # 完成当前 epoch
    trainer.finalize_epoch()
```

**等效关系**：
- `batch_size=16, accumulation_steps=4` 等效于 `batch_size=64`
- `batch_size=8, accumulation_steps=8` 等效于 `batch_size=64`

### 3. 自动批量大小调整


### 4. 混合精度训练（FP16）

使用 PyTorch 的自动混合精度训练降低显存使用：

```python
import torch
from torch.cuda.amp import autocast, GradScaler

# 创建梯度缩放器
scaler = GradScaler()

# 训练循环
for epoch in range(epochs):
    for X_batch, y_batch in dataloader:
        optimizer.zero_grad()
        
        # 使用自动混合精度
        with autocast():
            output = model(X_batch)
            loss = criterion(output, y_batch)
        
        # 缩放损失并反向传播
        scaler.scale(loss).backward()
        scaler.step(optimizer)
        scaler.update()
```

**优势**：
- 显存使用减少约 40-50%
- 训练速度提升约 2-3 倍
- 精度损失极小（< 0.1%）

## 常见问题与解决方案

### Q1: 训练时出现 "CUDA out of memory" 错误

**解决方案**：
1. 减小批量大小（batch_size）
2. 启用梯度累积保持有效批量大小
3. 使用混合精度训练（FP16）
4. 减小模型配置（hidden_dim, n_layers）
5. 清空 GPU 缓存：`torch.cuda.empty_cache()`

### Q2: 显存使用超过 20GB 警告

**解决方案**：
1. 检查是否有多余的中间变量占用显存
2. 使用 `del` 删除不需要的 Tensor
3. 在验证时使用 `torch.no_grad()` 上下文
4. 减小批量大小或模型配置

### Q3: 训练速度过慢

**解决方案**：
1. 启用混合精度训练（FP16）
2. 增大批量大小（如果显存允许）
3. 使用多 GPU 训练（DataParallel 或 DistributedDataParallel）
4. 优化数据加载（使用 DataLoader 的 num_workers 参数）
5. 使用 GPU 加速的数据预处理

### Q4: XGBoost 训练速度慢

**解决方案**：
1. 启用 GPU 加速：`tree_method='gpu_hist'`
2. 设置多线程：`n_jobs=-1`
3. 减小树深度：`max_depth=3` 或 `max_depth=4`
4. 增大学习率：`learning_rate=0.3`

## 最佳实践

### 1. 训练前检查

```python
from src.baselines.utils import get_gpu_memory_info

# 检查 GPU 状态
info = get_gpu_memory_info()
print(f"GPU 总显存: {info['total_gb']:.2f} GB")
print(f"可用显存: {info['free_gb']:.2f} GB")

# 确保有足够的可用显存（建议 > 10 GB）
if info['free_gb'] < 10.0:
    print("警告：可用显存不足，建议清理 GPU 缓存")
    torch.cuda.empty_cache()
```

### 2. 训练中监控

```python
with GPUMemoryMonitor(warning_threshold_gb=20.0) as monitor:
    for epoch in range(epochs):
        # 训练代码...
        
        # 每个 epoch 检查显存
        monitor.check_memory(context=f"Epoch {epoch+1}")
        
        # 如果显存使用过高，清空缓存
        stats = monitor.get_memory_stats()
        if stats['current_memory_gb'] > 18.0:
            torch.cuda.empty_cache()
```

### 3. 训练后清理

```python
# 删除模型和优化器
del model
del optimizer

# 清空 GPU 缓存
torch.cuda.empty_cache()

# 重置峰值显存统计
torch.cuda.reset_peak_memory_stats()
```
